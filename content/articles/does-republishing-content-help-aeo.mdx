**Republishing helps only when it reflects a genuine update on the same URL —
substantively revising a page and refreshing its date restores freshness while keeping
its authority.** Changing the URL or just bumping the date without real changes hurts
more than it helps, by resetting authority or sending a hollow signal.

## New URL or update in place?

Update in place, almost always. Keeping the same URL preserves the page's accumulated
[authority](/pillars/authority), links, and recognition, while republishing at a new
URL starts from zero and risks creating a [duplicate that competes with the
original](/learn/does-duplicate-content-hurt-aeo). So "republishing" should mean
refreshing the existing page, not moving it — the URL is an asset, and you don't want
to throw away what it's earned.

## Does changing the date count?

Not meaningfully. [Bumping the date without revising the
content](/learn/does-publish-date-affect-ai-citation) adds no real freshness and can
erode trust if the page is visibly stale — it's the hollow version of republishing.
Genuine republishing means [updating the substance](/learn/does-updating-old-content-help-aeo):
the date should *record* that work, not substitute for it. A date is only a signal of
freshness if real freshness is behind it.

## When is it worth doing?

When a page has decayed and a real update can restore it. If facts are stale,
developments are missing, or the answer can be improved, revise it substantively in
place and refresh the date — that recovers freshness on top of existing authority,
which is one of the [highest-return freshness
moves](/learn/cheapest-high-impact-aeo-moves). Done genuinely, it's just the refresh
half of the [Freshness pillar](/pillars/freshness); done hollowly, it's noise.
