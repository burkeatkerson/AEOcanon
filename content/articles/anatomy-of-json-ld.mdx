  - q: What are the main parts of a JSON-LD block?
    a: The four core parts are @context, which sets the vocabulary to schema.org; @type, which declares what the thing is; @id, which gives it a stable identifier; and the properties, which describe the entity. Nesting lets one entity reference or contain another.
  - q: What does @id do in JSON-LD?
    a: The @id property assigns a stable, unique identifier — usually a URL — to a node so other nodes can reference the same entity instead of duplicating it. It lets you link an Article to its author or publisher cleanly across your markup.

**A [JSON-LD](/glossary/json-ld) block has four load-bearing parts: `@context` sets the
vocabulary, `@type` declares what the thing is, `@id` gives it a stable identifier, and the
remaining properties describe it.** Learn these and you can write and debug any block.

  <p>

    entity is, <strong>@id</strong> gives it a <strong>stable identifier</strong> so other
    nodes can reference it, and <strong>properties</strong> fill in the details. Nesting lets
    one entity point to another.

## What does a real block look like?

Here's a compact Article with a nested author and publisher:

```json
{
  "@context": "https://schema.org",
  "@type": "Article",
  "@id": "https://aeocanon.com/learn/anatomy-of-json-ld#article",
  "headline": "The Anatomy of a JSON-LD Block",
  "author": {
    "@type": "Person",
    "@id": "https://aeocanon.com/authors/burke-atkerson#person",
    "name": "Burke Atkerson"
  },
  "publisher": {
    "@type": "Organization",
    "name": "AEO Canon"
  },
  "datePublished": "2026-06-30"
}
```

## How do I read each part?

Left to right, top to bottom — each key has a job.
